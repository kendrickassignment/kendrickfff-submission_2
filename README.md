# Telco Customer Churn Prediction — MLOps Pipeline

| | Deskripsi |
|---|---|
| Dataset | [IBM Telco Customer Churn](https://github.com/IBM/telco-customer-churn-on-icp4d) — 7.032 records pelanggan telekomunikasi dengan 13 fitur + 1 label |
| Masalah | Customer churn (kehilangan pelanggan) merupakan tantangan kritis di industri telekomunikasi. Biaya akuisisi pelanggan baru 5-7x lebih mahal dibanding mempertahankan pelanggan yang ada. Tanpa sistem prediksi yang akurat, tim product/marketing tidak dapat melakukan intervensi proaktif secara tepat sasaran sebelum pelanggan berhenti berlangganan. |
| Solusi machine learning | Membangun end-to-end machine learning pipeline menggunakan TFX (TensorFlow Extended) dengan model Deep Neural Network (DNN) untuk memprediksi probabilitas churn setiap pelanggan berdasarkan data profil dan perilaku. Pipeline dijalankan menggunakan Apache Beam orchestrator (BeamDagRunner) dan di-deploy sebagai REST API di cloud. Target: Binary Accuracy ≥ 78%, AUC ≥ 0.75. |
| Metode pengolahan | **Numerical features** (tenure, MonthlyCharges, TotalCharges): Z-score normalization menggunakan `tft.scale_to_z_score`. **SeniorCitizen**: Pass-through (sudah 0/1). **Categorical features** (gender, Partner, Dependents, PhoneService, InternetService, Contract, PaperlessBilling, PaymentMethod): Vocabulary encoding menggunakan `tft.compute_and_apply_vocabulary`. **Label** (Churn): Binary mapping Yes→1, No→0. Data di-split 80% train / 20% eval menggunakan ExampleGen. |
| Arsitektur model | Deep Neural Network (DNN) dengan arsitektur: Input Layer (numerical z-score + categorical embedding) → Dense(128, relu) + BatchNorm + Dropout(0.3) → Dense(64, relu) + BatchNorm + Dropout(0.3) → Dense(32, relu) + BatchNorm + Dropout(0.3) → Dense(1, sigmoid). Optimizer: Adam. Loss: Binary Crossentropy. Hyperparameter tuning otomatis menggunakan Keras Tuner (RandomSearch) melalui komponen Tuner. |
| Metrik evaluasi | **Binary Accuracy** (threshold ≥ 0.78) — mengukur akurasi klasifikasi keseluruhan. **AUC** (threshold ≥ 0.75) — mengukur kemampuan model membedakan kelas positif dan negatif. Evaluasi menggunakan TFMA (TensorFlow Model Analysis). |
| Performa model | Binary Accuracy: **0.808** (threshold 0.78 ✅ Pass). AUC: **0.870** (threshold 0.75 ✅ Pass). Model menunjukkan kemampuan sangat baik dalam membedakan pelanggan berisiko tinggi (churn probability 87.93%) dan rendah (churn probability 1.20%). |
| Opsi deployment | Model di-deploy menggunakan **TensorFlow Serving 2.11.0** dalam Docker container pada platform **Railway**. Model diakses melalui REST API (port 8501). Deployment dilakukan via GitHub integration — setiap push ke repository otomatis trigger build dan deploy. |
| Web app | [https://kendrickfff-submission2-production.up.railway.app/v1/models/churn-model](https://kendrickfff-submission2-production.up.railway.app/v1/models/churn-model) |
| Monitoring | Sistem dipantau menggunakan **Prometheus** yang scrape metrics dari TensorFlow Serving (request count, latency, model status). Prometheus disinkronkan dengan **Grafana** untuk dashboard monitoring visual. Hasil monitoring menunjukkan 3 target aktif: tf-serving, tf-serving-rest, dan prometheus. Dashboard Grafana menampilkan service status secara real-time. Monitoring stack dijalankan lokal menggunakan Docker Compose (`monitoring/docker-compose.yml`). |
